## Project Background

**Questionsfactory v1** is a single-file HTML MVP (7,695 lines, `index.html`) that trains question-asking ability using a 6-dimension framework (Purpose, Context, Boundary, Depth, Action, Assumption). It lives at: https://github.com/iks9245/Questionsfactory

**Questionsfactory v2** is the full B2B SaaS rebuild. Goal: make "prompt quality" measurable, trainable, and provably correlated with workplace AI productivity — and sell that proof to enterprises paying for Copilot/ChatGPT Enterprise licenses.

## Tech Stack (finalized)

| Layer | Choice | Reason |
|---|---|---|
| Monorepo | Turborepo + pnpm | Workspace dependency management |
| Frontend | Next.js 15 (App Router) + TypeScript | SSR, RSC, Vercel-native |
| UI | Tailwind CSS + shadcn/ui | Fast, consistent, accessible |
| State | Zustand + TanStack Query | Local UI state + server cache |
| Auth | Supabase Auth | Built-in JWT, org-level RLS |
| DB | Supabase PostgreSQL | RLS for multi-tenancy, Realtime |
| Edge Functions | Supabase Edge Functions (Deno) | LLM scoring without cold starts |
| Extension | Chrome Extension MV3 + React | Content script injection into AI tools |
| Deployment | Vercel (web) + Chrome Web Store (ext) | Zero-config deploys |

## Core Scoring Algorithm — TypeScript Interfaces

```typescript
// packages/scoring/src/types.ts

interface DimensionScores {
  purpose:    number  // 0-100: Is the goal clear?
  context:    number  // 0-100: Is background provided?
  boundary:   number  // 0-100: Are scope/constraints defined?
  depth:      number  // 0-100: Is reasoning depth appropriate?
  action:     number  // 0-100: Is the desired action explicit?
  assumption: number  // 0-100: Are assumptions stated?
}

interface ScoringResult {
  dimensions: DimensionScores
  overall: number          // weighted average
  weakest: string[]        // dimensions below 60
  suggestions: string[]    // actionable feedback strings
  mode: 'rule-based' | 'llm'
  confidence: number       // 0-1, higher for llm mode
}

interface PromptLogEntry {
  id: string
  user_id: string
  org_id: string
  platform: 'chatgpt' | 'claude' | 'copilot' | 'gemini'
  scores: DimensionScores
  overall_score: number
  prompt_length: number      // character count only, NOT raw text
  satisfaction_rating: 1 | 2 | 3 | 4 | 5 | null
  created_at: string
}
```

## Key Design Decisions

### Chrome Extension
- **Inject into**: ChatGPT (chat.openai.com), Claude (claude.ai), GitHub Copilot Chat, Gemini (gemini.google.com)
- **Scoring trigger**: User presses Send button — score the textarea content locally before submit
- **Privacy**: Never send raw prompt text to server. Only send `DimensionScores + prompt_length + platform`
- **Satisfaction capture**: Show a 5-star modal 30 seconds after AI responds (non-blocking, dismissible)
- **Overlay UI**: Score badge appears bottom-right of the chat input, showing overall score and weakest dimension

### Multi-tenancy (Supabase RLS)
```sql
-- All tables use org_id for isolation
-- Example: users can only see teammates
CREATE POLICY "users_org_isolation" ON users
  FOR ALL USING (
    org_id = (SELECT org_id FROM users WHERE id = auth.uid())
  );
```

### Dual Scoring Modes
- **Rule-based** (packages/scoring/src/rule-based.ts): runs in Extension content script, 0ms latency, no API cost, pattern-matching heuristics
- **LLM-backed** (Supabase Edge Function score-prompt): used in Training module for detailed coaching feedback, costs ~$0.001/call, called only when user explicitly trains

## B2B Model Summary

- **Target**: 50-300 person tech/consulting companies already paying for Copilot or ChatGPT Enterprise
- **Free Pilot**: 3 months free, company provides anonymized usage data as payment
- **Paid**: $25/user/month (Team), $40/user/month (Enterprise, 100+ users)
- **Value prop**: Prove that higher prompt scores → higher AI satisfaction → better output quality

## Where to Start (First Session)

1. Run `pnpm create turbo@latest` to scaffold the monorepo
2. Add `apps/web` (Next.js 15), `apps/extension` (Vite + React), `packages/scoring`
3. Set up Supabase project, run migrations from `docs/04-database-schema.md`
4. Implement `packages/scoring/src/rule-based.ts` — this is the core IP
5. Build the Chrome Extension content script (start with ChatGPT only)
6. Build the personal dashboard in `apps/web` — ability profile radar chart

Follow the 90-day roadmap in `docs/05-roadmap.md`.
