# Questionsfactory

Questionsfactory 是一個用來訓練「提問能力」的單頁 MVP。

公開頁面：<https://iks9245.github.io/Questionsfactory/>

這個項目相信一件事：在 AI 盛行的時代，答案會變得越來越容易取得，但真正決定思考品質的，仍然是問題本身。

## 項目目標

Questionsfactory 先訓練使用者判斷什麼是好問題，再逐步練習把普通問題升級成更清晰、更有行動價值的問題。

目前首頁已改成單頁多視圖 App：保留單一 `index.html` 的傳播優勢，但把內容分成「訓練、進度、能力、題庫」四個工作區，避免所有功能擠在同一條長頁上。

目前的學習循環是：

1. 判斷四個選項中哪一個問題品質最高
2. 看見好問題背後的評分維度
3. 練習改寫較弱的問題
4. 讓 AI 問題教練評改自己的改寫版本
5. 透過個人化訓練路徑完成今日練習
6. 透過固定題庫或 LLM 生成題庫持續練習

## 核心維度

題目會圍繞六個問題品質維度：

- 目的：問題是否知道自己要解決什麼
- 情境：是否提供足夠背景
- 邊界：是否收斂範圍與限制
- 深度：是否能推動分析，而不是只要表面答案
- 行動：是否能導向下一步
- 假設：是否能暴露前提與盲點

## 功能

- 單一 `index.html`，不需要打包或後端
- 單頁多視圖導航：訓練、進度、能力、題庫分區操作
- 內建離線起始題庫
- 支援一次生成 10 至 30 條 LLM 題庫
- 支援題目難度：入門、中階、高階，並可由能力模型自動調整
- 支援雲端 LLM：
  - OpenAI Responses
  - Google Gemini
  - Anthropic Claude
  - OpenRouter
  - OpenAI-compatible endpoints
- 支援本機 LLM：
  - OMLX / MLX server
  - Local OpenAI-compatible servers
  - Ollama native API
- 題目難度與品質校準引擎 2.0，降低「答案越長分數越高」這類明顯破綻
- 低品質候選題會自動丟棄，不足 10 題時會重試補題
- 可修正題會自動補足評分理由與校準分數
- 題庫管理：保存、刪除、編輯題目與分數
- JSON 匯出與匯入，方便備份或搬運題庫
- AI 問題教練：根據六個維度評改使用者提交的改寫問題
- 教練會提供具體診斷、1 至 3 個追問、AI 示範改寫與三版本比較
- 改寫評估紀錄會保存在瀏覽器 `localStorage`
- 個人化問題能力模型：合併辨識題成績與 AI 教練評分，找出最弱維度與趨勢
- 錯題復盤管理：答錯辨識題或改寫低分時，自動記錄錯因，並可篩選、清除或用最近錯因開啟訓練
- 自適應訓練閉環：今日訓練與弱項題庫生成會優先回補最近常犯錯因
- 一鍵生成弱項題庫，針對目的、情境、邊界、深度、行動、假設中的薄弱項訓練
- 個人化訓練路徑：一鍵開始今日 10 分鐘訓練，串起辨識題、改寫題與 AI 教練評估
- 今日訓練可選自動、入門、中階或高階難度；自動模式會根據能力模型調整
- 訓練 session 會保存小結，並可查看最近 7 次訓練紀錄
- 7 天訓練週期：顯示每日訓練狀態、連續完成天數、本週完成天數與下一次推薦
- 成效曲線與六維趨勢：用最近 7 天 session 看見模型分與各維度變化
- 分享入口與 GitHub Pages 公開頁面
- 使用瀏覽器 `localStorage` 保存練習進度

## 頁面結構

Questionsfactory 目前保留單頁部署，但不再把所有內容直線堆疊。

- 訓練：預設首頁，聚焦今日 10 分鐘訓練、本週簡表、辨識題與改寫題
- 進度：集中查看 7 天訓練週期、streak、成效曲線與六維趨勢
- 能力：查看問題能力模型、診斷、教練紀錄與分享入口
- 題庫：集中管理 LLM 出題、Local LLM 端點、題庫編輯、匯出與匯入

這個結構的目標是讓新使用者第一眼就能開始訓練；進階設定與歷史資料則放到需要時才打開。

## 接入 Local LLM

如果使用 OMLX / MLX server，可以在頁面中選擇：

```text
Provider: OMLX / MLX server
Endpoint: http://127.0.0.1:11123/v1
```

頁面會自動把 base URL：

```text
http://127.0.0.1:11123/v1
```

轉成 OpenAI-compatible chat completions 路徑：

```text
http://127.0.0.1:11123/v1/chat/completions
```

如果本機服務要求 API key，請填入 API key。若 `curl` 可以成功但瀏覽器失敗，通常是本機模型服務沒有開啟 CORS。

## AI 問題教練

在「改寫訓練」區輸入自己的改寫版本後，可以按「AI 教練評估」。

AI 教練會共用「AI 出題」區的模型設定，所以同時支援：

- OpenAI Responses
- Google Gemini
- Anthropic Claude
- OpenRouter
- OpenAI-compatible endpoints
- OMLX / MLX server
- Local OpenAI-compatible servers
- Ollama native API

教練回饋包含：

- 整體 0 至 100 分
- 目的、情境、邊界、深度、行動、假設六維度各 0 至 5 分
- 每個維度的具體診斷與改進建議
- 1 至 3 個追問，幫使用者補足缺失資訊
- 一個 AI 示範改寫版本
- 「原弱問題 / 我的改寫 / AI 示範改寫」三版本比較

如果沒有可用模型，也可以使用「本地快速評分」。本地評分不會呼叫 API，但仍會保存到教練紀錄，方便回看練習軌跡。

## 個人化問題能力模型

側欄的「能力模型」會把兩種資料合併：

- 辨識訓練：你選中的候選問題在六個維度上的分數
- AI 問題教練：你自己改寫問題後得到的六維評分

模型會顯示：

- 目前模型分
- 最弱的 1 至 2 個維度
- 每個維度的辨識證據數與教練證據數
- 最近進步、穩定、下滑或資料累積狀態
- 個人化診斷與下一步建議

能力頁也會顯示「錯題復盤」。系統會把兩類資料整理成常犯錯因：

- 辨識題答錯：比較使用者選項與最佳選項的六維差距
- 改寫題低分：擷取 AI 教練評分中最低的維度

每筆復盤會保存：

- 錯題來源與分數
- 最相關的 1 至 2 個錯因維度
- 為什麼這個錯誤選項有迷惑性
- 下一次改寫或判斷時該補的具體結構

按「生成弱項題庫」時，系統不只看目前最弱維度，也會把最近常犯錯因寫進 LLM 題目主題，並呼叫你在「AI 出題」區選擇的供應商生成針對性題庫。

錯題復盤也可以直接管理：

- 用「全部 / 辨識錯 / 教練低分」篩選復盤紀錄
- 按「用錯因訓練」直接根據最近常犯錯因開始一輪今日訓練
- 按「清除復盤」清空錯題復盤資料，重新累積錯因

## 個人化訓練路徑

首頁的「今日 10 分鐘訓練」會根據能力模型安排一輪訓練。

每次訓練包含三步：

1. 辨識題：先選出本題最佳問題，校準判斷力
2. 改寫題：把同一題的弱問題改成更可行動的版本
3. AI 教練評估：用同一組 LLM 設定做六維診斷與追問

完成後，系統會保存這次 session，並顯示：

- 本次訓練目標
- 辨識題是否答對與選項分數
- AI 教練評分來源與分數
- 本次有進步的維度
- 仍需加強的維度

最近 7 次 session 會保存在瀏覽器 `localStorage`，方便回看訓練節奏。

訓練難度可以手動選擇，也可以使用「自動」：

- 入門：答案差異較清楚，適合建立六維判斷基礎
- 中階：四個選項都合理，靠細節維度分出勝負
- 高階：干擾項也很強，答案不能靠長度、語氣或分數猜出

自動模式會根據目前能力模型與證據量調整。資料少或模型分偏低時先用入門題；能力穩定後進入中階；模型分與證據量都足夠時才推高階題。

如果最近有錯題復盤，今日訓練會優先安排常犯錯因對應的維度。例如使用者近期常在「邊界」和「假設」失分，系統會優先挑選能拉開這兩個維度差距的辨識題，讓訓練從「平均補弱」變成「針對最近錯誤回補」。

## 7 天訓練週期

首頁的「本週訓練概況」會把最近 7 天整理成一個週期視圖。

它會顯示：

- 最近 7 天每天是否完成訓練
- 當天安排或完成的訓練重點
- 連續完成天數 streak
- 本週完成天數
- 本週最進步維度
- 目前仍最弱的維度
- 最近 7 天模型分成效曲線
- 六個能力維度的趨勢變化
- 下一次訓練方向建議

每日訓練重點不是固定題庫順序，而是根據能力模型與日期做輪替。系統會優先從較弱、證據較少或需要校準的維度中安排 1 至 2 個重點，讓使用者每天回來時都有不同的訓練焦點。

## 題目難度與品質校準 2.0

LLM 生成題目時，頁面會先要求模型多產生候選題，再由本地品質引擎篩選。

出題時可以選擇：

- 自動：根據能力模型決定入門、中階或高階
- 入門：最佳選項可以比較清楚，但仍不能靠字數取勝
- 中階：四個選項都要像成熟問題，最佳選項只靠一到兩個關鍵結構勝出
- 高階：三個干擾項都必須有真實優點，只缺一個細微但決定性的結構

每一題都會檢查：

- 是否剛好有四個候選問題
- 四個選項長度是否接近
- 最佳選項是否明顯更長
- 分數差距是否過大
- 干擾選項是否太弱
- 是否有足夠接近最佳答案的強干擾項
- 六維總分差距是否讓答案過於明顯
- 題目難度是否符合本輪目標
- 每個選項是否有清楚評分理由
- 題目解析是否說明最佳選項為什麼勝出

系統會盡量避免：

- 最佳答案永遠最長
- 分數差距過大
- 干擾選項明顯很弱
- 高階題只有一個像樣選項
- 題目只是在考文字長度，而不是問題品質

如果候選題只是分數、理由或難度標記不夠好，頁面會做本地修正；如果選項長度差太多、最佳答案太明顯、干擾項太弱，或高階題缺少強干擾項，該題會被丟棄。

當保留題目不足 10 題時，頁面會把上一輪的品質問題回饋給模型，最多自動重試 3 輪補題。生成完成後，頁面會顯示品質報告，包括目標難度、候選數、保留數、修正數、重試輪次、保留難度分布與主要問題。

## 題庫管理

LLM 生成的題庫會自動保存到瀏覽器 `localStorage`。在「題庫管理」區可以：

- 選擇已保存的 AI 題目
- 編輯題目標題、場景、弱問題、四個候選問題、分數、評分理由與解析
- 刪除單題
- 匯出 JSON 備份
- 重新匯入 Questionsfactory JSON 題庫

匯出的 JSON 結構包含：

```json
{
  "app": "Questionsfactory",
  "version": 1,
  "exportedAt": "2026-06-17T00:00:00.000Z",
  "lessons": []
}
```

## 隱私說明

API key 是在瀏覽器端輸入，並由瀏覽器直接送到你選擇的模型服務。

預設不會保存 API key。只有在勾選「記住 API key」時，key 才會以純文字存在這台瀏覽器的 `localStorage`。

如果要公開部署或給團隊使用，建議改成後端 proxy，由後端保管 API key。
